# Case Study 5 - Splunk Searches

## 1. Show all Registry Value Set events

```spl
index=* EventCode=13
```

**Purpose:**
Displays every registry value modification event collected by Sysmon.

---

## 2. Display important investigation fields

```spl
index=* EventCode=13
| table _time Image TargetObject Details User
```

**Purpose:**
Shows the event time, process, registry path, registry value, and user account.

---

## 3. Find the process responsible for creating the Registry Run Key

```spl
index=* EventCode=1 reg.exe
```

**Purpose:**
Displays the execution of **reg.exe**, which was used to create the persistence entry.

---

## 4. Show Registry Value Set events created by reg.exe

```spl
index=* EventCode=13 Image="*reg.exe"
| table _time Image TargetObject Details User
```

**Purpose:**
Filters registry modification events to only those generated by **reg.exe**, making it easier to identify the persistence activity.

---

## 5. Count Registry Value Set events by process

```spl
index=* EventCode=13
| stats count by Image
```

**Purpose:**
Shows which processes modified registry values.

---

## 6. Count Registry Value Set events by user

```spl
index=* EventCode=13
| stats count by User
```

**Purpose:**
Identifies which user account performed registry modifications.

---

## 7. Create a timeline of Registry Value Set events

```spl
index=* EventCode=13
| timechart count
```

**Purpose:**
Visualizes registry modification activity over time.

---

## 8. Search for the TestPersistence Registry value

```spl
index=* EventCode=13 TestPersistence
```

**Purpose:**
Searches specifically for the Registry Run Key created during the lab.

> **Note:** If this query returns no results because of your Sysmon configuration, use Query 2 or Query 4 to locate the event manually.
